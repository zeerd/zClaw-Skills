# Article Analyzer

专业的内容分析技能，帮助深度解读文章的核心价值。

**工作流程**：
1. 用户提供链接 → 我调用 `web-content-fetcher` 技能获取内容
2. 对于微信公众号文章，`web-content-fetcher` 会使用 Scrapling 绕过反爬
3. 获取内容后，按五层框架进行分析

**注意**：请直接提供链接给我，不要自己先调用 `web_fetch` 获取内容。

## 使用场景

当你看到高质量的文章，想要借助 AI 进行深入分析时使用此技能：

- 分析文章的核心论点和论证结构
- 理解作者的背景、立场和写作意图
- 批判性审视观点的合理性和局限性
- 提取可复用的思考框架和方法论
- 分析写作技巧和说服策略

## 分析框架

按照以下五层框架逐层回答问题：

### 一、核心内容（搞清楚"是什么"）

1. 文章的核心论点是什么？用一句话概括
2. 作者用了哪些关键概念？这些概念是怎么定义的？
3. 文章的结构是什么？论证是怎么展开的？
4. 有哪些具体案例或证据支撑观点？

### 二、背景语境（理解"为什么"）

1. 作者是谁？他的背景、身份、立场是什么？
2. 这篇文章是在什么背景下写的？在回应什么现象或争论？
3. 作者想解决什么问题？想影响谁？
4. 作者的底层假设是什么？有哪些没说出来的前提？

### 三、批判性审视

1. 有人会怎么反驳这个观点？主要的反对意见可能是什么？
2. 作者的论证有没有漏洞、跳跃或偏颇之处？
3. 这个观点在什么情况下成立？什么情况下不成立？边界在哪里？
4. 作者有没有刻意回避或淡化什么问题？

### 四、价值提取

1. 作者提出了什么可复用的思考框架或方法论？
2. 对于目标读者角色，能从中学到什么？
3. 这篇文章可能改变读者的什么认知？

### 五、写作技巧分析（可选）

1. 文章的标题、开头、结尾是怎么设计的？
2. 作者用了什么技巧让文章有说服力？
3. 这篇文章的写法有什么值得学习的地方？

## 使用方法

### 基本用法

接收提供给我的文章内容，按照上述框架进行深度分析：

```
请分析这篇文章：
[粘贴文章内容或提供链接]
```

✅使用`web-content-fetcher`技能处理链接。
❌禁止使用`web_fetch`读取`mp.weixin.qq.com`的文章。

### 自定义分析

指定目标读者角色，让分析更有针对性：

```
请分析这篇文章，特别关注对于 [产品经理/开发者/创业者] 的价值：
[文章内容]
```

### 特定问题

如果只关注某些方面，可以指定：

```
请重点分析这篇文章的论证逻辑和可复用的方法论：
[文章内容]
```

## 输出格式

分析结果将包含：

- **核心论点**：一句话概括
- **关键概念**：清晰定义的术语
- **论证结构**：逻辑展开的脉络
- **背景洞察**：作者的意图和立场
- **批判性观点**：可能的反驳和局限性
- **价值总结**：可复用的框架和方法
- **写作亮点**：值得学习的技巧
- **原文链接**：如果文章是以链接的方式提供的

回答要具体、有洞察，避免泛泛而谈。如果某个问题信息不足无法回答，会说明原因。

## 注意事项

- 支持分析长文，如果内容过长可以分段提供
- 可以提供文章链接，我会尝试获取内容
- 可以指定多个目标读者角色，获得不同视角的洞察
- 分析结果可以直接用于写作、分享或深度思考
