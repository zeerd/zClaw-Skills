# 小说编写技能

## 🚀 工作流程

使用 **渐进式披露设计**：核心工作流在 SKILL.md，详细技术指南在 [references/workflow_guide.md](references/workflow_guide.md)。

```
用户输入创作资料
    ↓
[Step 1] 读取并分析输入资料
    ↓
[Step 2] 梳理章节大纲 (文档 1)
    ↓
[Step 3] 创建人物侧写 (文档 2)
    ↓
[Step 4] 开始逐章编写 (文档 3)
    ↓
持续更新三类文档直至完成
```

📚 **详细工作流指南**: 见 [references/workflow_guide.md](references/workflow_guide.md)，包含：
- 阶段一：资料收集与分析（输入资料分类、分析步骤、澄清问题模板）
- 阶段二：章节大纲设计（结构框架、章节划分原则、模板）
- 阶段三：人物侧写构建（档案模板、关系图谱）
- 阶段四：正文撰写规范（段落结构、写作风格、章节长度）
- 阶段五：迭代与修订（反馈处理、常见问题诊断）

## 三文档系统设计

这个技能会首先在工作区创建一个新的文件夹，命名为"novels/[小说名称]"，即`~/.openclaw/workspace/novels/[小说名称]`。在这个文件夹内创建或维护**三类独立文档**：

### 📄 文档 1：需求与大纲

**用途**：存储所有创作规划材料，作为"小说蓝图"，命名为"需求与大纲.md"。

**包含内容**：
- 核心设定与世界观
- 创作需求分析
- 章节大纲与剧情走向
- 情节结构图
- 关键转折点标注
- 翻译（如果有资料非中文，首先将它们翻译成中文并保存起来）

**更新策略**：每次迭代后更新，保留修改痕迹

**版本管理**（参考 [workflow_guide.md](references/workflow_guide.md)）：
- 首次创建：v1.0
- 大纲调整：v1.1, v1.2...
- 大幅修订：v2.0, v3.0...

**文件命名建议**：`[小说名]_需求与大纲_vX.md`

### 📄 文档 2：人物档案与侧写

每个人物（主角、配角、反派等）一个独立文档，命名为"角色-[角色姓名].md"，存储该角色的详细信息和发展轨迹。

**用途**：详细记录每个角色的特征、背景、动机等信息

**包含内容**：
- 基础信息（姓名、年龄、外貌等）
- 性格特质（优点、缺点、信仰、恐惧等）
- 背景故事（童年经历、关键转折点、当前处境）
- 动机系统（外在目标、内在需求、行为驱动力、阻碍因素）
- 能力与限制（核心技能、知识领域、资源、人脉、身体/精神限制）
- 语言特征（说话风格、口头禅、语气特点、特殊表达习惯）
- 关系网络（与其他角色的关系类型和特点）

详见[人物侧写模板](references/character_template.md)

**更新策略**：每当要开始编写新的章节之前，根据本章中将要登场的新角色，每新增一个角色或对现有角色进行重要修改时更新

### 📖 文档 3：正文连载

每个章节一个独立文档，命名为"章节-[章节编号]-[章节标题].md"，存储该章节的完整正文内容。以免内容过长导致编辑器卡顿操作失误，必须每章单独一个文档。

**用途**：实际的小说正文内容

**包含内容**：
- 章节正文
- 场景描写与对话
- 人物动作与心理刻画
- 风格统一的叙事语言
- 全中文输出

**更新策略**：每完成一章新建下一个文档，保持叙事连贯性

## 开始创作前需要准备

请提供以下信息（尽可能详细）：

### 🔹 基础信息
- **小说类型**：如玄幻、科幻、悬疑、言情、历史、武侠等
- **目标文体**：网络小说风格、传统文学、轻小说、现实主义等
- **预计篇幅**：短篇 (<10 章)、中篇 (10-30 章)、长篇 (>30 章)、系列连载

### 🔹 核心设定
- **故事梗概**：用 100-300 字简述故事主线
- **世界观背景**：时代背景、社会规则、特殊设定
- **主题立意**：想表达的核心思想或情感基调

### 🔹 人物要素
- **主角设定**：姓名、年龄、性格、背景、动机
- **配角角色**：关键配角及其关系
- **反派/冲突**：主要对立面的特征

### 🔹 参考风格
- **喜欢的作者/作品**：希望学习或参考的写作风格
- **语言特点**：幽默、严肃、诗意、冷峻、口语化等
- **特殊要求**：需要遵循的格式、禁忌或特定元素

## 工作流程详解

📚 **完整技术指南**: [references/workflow_guide.md](references/workflow_guide.md)

详细工作流程分为五个阶段：

### 阶段一：资料收集与分析
- 核心资料（必需）、增强资料（推荐）、可选资料（推荐）
- 资料分析步骤（提取要素、发现矛盾、标注缺失、评估可行性、翻译）
- 澄清问题模板（5 个核心问题）

### 阶段二：章节大纲设计
- 结构框架（三幕/四幕结构）
- 章节划分原则（节奏控制、命名规范）
- 章节大纲模板（核心内容、关键场景、字数建议、衔接说明）

### 阶段三：人物侧写构建
- 人物档案模板（基本信息、核心动机、背景故事、关系网络、语言特征、成长弧光）
- 角色关系图谱（亲情、友情、爱情、敌对、师徒、利用等）

### 阶段四：正文撰写规范
- 段落结构建议（开场钩子、主体、结尾）
- 写作风格指南（叙事、对话、场景）
- 章节长度控制（标准 2000-3000 字、短章、长章）

### 阶段五：迭代与修订
- 反馈处理流程（4 步流程）
- 常见问题诊断（剧情、人物、语言问题）

---

### Step 1: 读取并分析资料

**我会**：
1. 仔细阅读你提供的所有资料
2. 识别关键信息和遗漏点
3. 提出澄清问题（如果有必要）
4. 总结核心创作要素供你确认

**输出**：创作资料分析报告

---

### Step 2: 梳理章节大纲

**我会**：
1. 根据核心设定设计整体结构
2. 规划章节划分与剧情节奏
3. 标注关键情节点（开场、激励事件、转折点、高潮、结局）
4. 为每章撰写简短梗概

**输出**：完整章节大纲，包含：
- 章节编号与标题建议
- 每章核心内容描述
- 章节字数建议
- 悬念与钩子设计

---

### Step 3: 创建人物侧写

**我会**：
1. 分析所有登场人物（主角、配角、反派等）
2. 为每个角色创建独立档案
3. 标注角色关系网络和成长弧光
4. 设计角色的语言特征和行为习惯
5. 确保角色设定满足"高一致性人脸"需求（如果涉及视觉描述）

**人物侧写包含**：
- **基础信息**：姓名、年龄、身份、外貌特征
- **性格特质**：核心性格、优点、缺点、信仰、恐惧
- **背景故事**：过往经历、关键转折点、动机与目标
- **语言风格**：说话方式、口头禅、语气特点
- **角色关系**：与其他角色的互动模式
- **成长轨迹**：从开始到结束的变化曲线

---

### Step 4: 开始逐章编写

**我会**：
1. 从第 1 章开始撰写完整正文
2. 每章 2000-3000 字（或者按着你的要求调整）
3. 兼顾场景描写、对话、内心活动
4. 保持叙事节奏和风格统一
5. 每章结尾设置悬念或钩子（适合连载）
6. 每章完成后自查，确保内容连贯、角色行为一致、风格统一
7. 检查是否存在内容重复、占位符、逻辑漏洞等问题

**每章完成后**：
- 如有需要，更新文档 2 的人物成长记录
- 第 1 章完成后，等待你确认或提出修改建议，再继续下一章；后续章节连续完成不再等待

## 创作风格指南

详细的**写作风格指南**参考 [references/workflow_guide.md](references/workflow_guide.md) 的"阶段四：正文撰写规范"。

### 叙事技巧
- **"Show, Don't Tell"**：通过动作、对话、细节展现人物和情节
- **场景转换**：使用清晰的空间/时间标记，避免场景跳跃
- **对话设计**：每个人物有独特说话方式，避免"信息倾销"式对话
- **节奏控制**：根据剧情张力调整段落长度和句长
- **信息隐藏**：适当保留信息，制造悬念，避免在人物登场时一次性暴露全部背景、能力。同样的，重要信息不要在对话中直接说出，而是通过行为、暗示、环境描写等方式展现。
- **情景互动**：避免大量的纯粹的对话内容。如果一定要通过对话推进剧情，可以适当安排一些情景互动，比如在房间或开放场景内各处走动，与周边的事物进行互动。比如壁画上的隐喻、或者户外的墙体、柴垛、植被花圃等。

### 角色塑造
- 每个重要角色至少有**2-3 个鲜明特征**
- 对话和动作要符合角色背景和性格
- 通过冲突展现角色深度
- 避免脸谱化，塑造复杂立体的人物

### 世界观呈现
- **渐进式揭示**：不要一次性堆砌设定
- **融入叙事**：通过角色经历自然展示世界观
- **规则一致性**：确保设定的逻辑自洽

## 用户协作方式

### 我可以如何协助你？

1. **创作咨询**：讨论剧情发展、人物选择、情节转折
2. **问题诊断**：分析卡点，提供解决方案
3. **风格调整**：根据你的反馈调整写作风格
4. **续写/扩写**：基于你的大纲继续创作
5. **修改建议**：对已完成内容提出优化意见
6. **多版本生成**：为特定场景提供不同写法供选择

### 常见创作场景

**场景 1：从零开始创作**
```
你：我想写一个关于末世重生的故事...
我：好的，我们来梳理核心设定和大纲。
```

**场景 2：卡文时需要灵感**
```
你：第 7 章卡住了，主角面临道德困境，不知道怎么写...
我：我提供 3 个不同的处理方式供你选择。
```

**场景 3：角色发展困惑**
```
你：反派好像立不住，总是为了坏而坏...
我：我们重新设计反派的动机和背景故事。
```

**场景 4：需要修改风格**
```
你：这一章写得太严肃了，想要更轻松搞笑...
我：我调整语言和节奏，重写这一章。
```

## 最佳实践建议

### ✅ Do
- 在开始创作前尽量提供详细的设定
- 每完成一章后给出反馈
- 及时确认人物侧写是否准确
- 明确标注不可修改的设定

### ❌ Don't
- 设定前后矛盾
- 突然改变已确定的关键设定
- 等待所有设定完美才启动（可以迭代完善）
