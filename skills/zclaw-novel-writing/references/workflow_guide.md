# 小说编写工作指南

本指南详细说明了技能如何执行小说创作流程，以及双文档管理的具体策略。

## 工作流程详解

### 阶段一：资料收集与分析

#### 1.1 输入资料分类

**核心资料（必需）**：
- 小说类型
- 故事梗概
- 主角基本信息
- 目标读者/风格定位

**增强资料（推荐）**：
- 世界观设定
- 配角列表
- 主题立意
- 篇幅规划
- 参考作品

**可选资料**：
- 完整大纲
- 详细人物档案
- 特定场景设计
- 禁忌元素列表

#### 1.2 资料分析步骤

1. **提取关键要素**：识别故事的核心冲突、主角动机、世界观规则
2. **发现矛盾点**：检查设定是否存在逻辑冲突或不一致
3. **标注缺失项**：标记需要进一步澄清的信息
4. **评估可行性**：根据提供信息判断创作难度和建议篇幅
5. **资料翻译**：将所有非中文资料翻译成中文（如果存在）

#### 1.3 澄清问题模板

当信息不足时，可以询问：

```
【创作要素澄清】
1. 关于【核心设定】：你是否希望...
2. 关于【角色塑造】：主角是否应该有...
3. 关于【剧情节奏】：你更喜欢...
4. 关于【结局走向】：你希望故事...
```

### 阶段二：章节大纲设计

#### 2.1 结构规划框架

**经典三幕结构**（适合大多数长篇）：
- 第一幕（1-25%）：铺垫、激励事件、转折一
- 第二幕（25-75%）：冒险、考验、挫折、转折二
- 第三幕（75-100%）：高潮、结局

**四幕结构**（适合节奏较快的网文）：
- 第一幕：铺垫与激励
- 第二幕：上升行动
- 第三幕：危机与低谷
- 第四幕：高潮与解决

#### 2.2 章节划分原则

**节奏控制**：
- 爽文：每 3-5 章一个小高潮，每 10-15 章一个大高潮
- 正剧：注重起承转合，节奏相对平缓
- 悬疑：持续埋设线索，定期揭晓谜团

**章节命名**：
- 第 1 章：必须制造悬念或钩子
- 中间章节：可考虑用关键词提示剧情
- 高潮章节：标题要有张力和冲击力

#### 2.3 章节大纲模板

```markdown
## 第 X 章：[章节标题]

### 核心内容
- 剧情推进：本章要完成的核心事件
- 情感重心：主角或关键角色的情感变化
- 悬念设计：本章结尾留下的问题或期待
- 角色成长：角色在本章的变化或新认知

### 关键场景
1. [场景 1]：发生地点、主要人物、核心冲突
2. [场景 2]：发生地点、主要人物、核心冲突

### 字数建议
[2000-3000] 字

### 衔接说明
- 上章关联：如何承接上一章结尾
- 下章铺垫：如何为下一章埋线
```

### 阶段三：人物侧写构建

#### 3.1 人物档案模板

```markdown
## [角色姓名]

### 基本信息
- **年龄**：
- **身高/体型**：
- **外貌特征**：
- **身份/职业**：
- **性格标签**：3-5 个关键词

### 核心动机
- **外在目标**：角色想实现的具体目标
- **内在需求**：角色需要的情感或心理需求
- **行为驱动力**：什么会促使角色行动

### 背景故事
- **童年/早期经历**：塑造性格的关键事件
- **转折点**：改变角色人生轨迹的事件
- **遗憾/秘密**：角色不愿提及的事

### 关系网络
- **与主角**：[关系类型及特点]
- **与配角**：[重要互动]
- **对立面**：[对立人物及原因]
- **盟友**：[支持者]

### 语言特征
- **说话风格**：正式/口语/诗意/简练
- **口头禅**：
- **语气特点**：

### 成长弧光
- **起点特质**：故事开始时
- **转变契机**：什么事件促使变化
- **终点状态**：故事结束时
```

#### 3.2 角色关系图谱

推荐使用以下格式记录关键关系：

```
[角色 A] ──[关系类型]──> [角色 B]
```

关系类型包括：
- **亲情**：父子、母女、兄弟姐妹
- **友情**：挚友、同窗、伙伴
- **爱情**：恋人、暧昧、单相思
- **敌对**：宿敌、竞争对手、对立阵营
- **师徒**：教导关系
- **利用**：互相利用、背叛

### 阶段四：正文撰写规范

#### 4.1 段落结构建议

**开场段落（钩子）**：
- 用场景、动作或悬念吸引读者
- 避免冗长的背景介绍
- 建立读者兴趣

**主体段落**：
- 每个段落有明确功能（推进剧情、塑造人物、展示世界观）
- 对话和叙述交替出现，避免单调
- 适当使用短句制造张力

**结尾段落**：
- 完成本章核心内容
- 提供悬念或情感满足
- 为下一章制造期待

#### 4.2 写作风格指南

**叙事语言**：
- 根据题材选择正式程度
- 保持统一的语调
- 避免过度使用形容词

**对话设计**：
- 每人说话方式不同
- 避免"对话即解释"
- 用对话展现冲突和性格

**场景描写**：
- 调动五感（视觉、听觉、嗅觉、触觉）
- 避免堆砌描写影响节奏
- 通过环境烘托氛围

#### 4.3 章节长度控制

**标准章节**：2000-3000 字
- 适合网络连载，节奏适中
- 阅读时间约 10-15 分钟

**短章节**：1500-2000 字
- 适合快节奏网文
- 每章制造一个小的完成感

**长章节**：3000-4000 字
- 适合重大情节节点
- 适合高潮章节或情感戏

---

### 阶段五：迭代与修订

#### 5.1 反馈处理流程

1. **用户给出具体反馈**→分析反馈指向（剧情、人物、语言）
2. **提供修改方案**→给出 2-3 个调整选项
3. **执行修改**→根据选择进行修改
4. **对比原稿**→保留修改痕迹或说明改动原因

#### 5.2 常见问题诊断

**剧情问题**：
- 节奏拖沓 → 删减或合并场景
- 突兀转折 → 增加铺垫
- 缺乏冲突 → 强化对立立场

**人物问题**：
- 角色扁平 → 增加背景故事和动机
- 行为不一致 → 回顾人物设定调整
- 对话失真 → 根据角色特征重写

**语言问题**：
- 风格不稳 → 调整用语习惯
- 描写不足 → 增加环境或心理描写
- 过于口语 → 调整文风正式度
- 逻辑断裂 → 作好铺垫

## 双文档管理策略

### 文档 1：需求与大纲

**文件命名建议**：
```
[小说名]_需求与大纲_vX.md
```

**版本管理**：
- 首次创建：v1.0
- 大纲调整：v1.1, v1.2...
- 大幅修订：v2.0, v3.0...

**更新时机**：
- 核心设定确定后
- 章节大纲完成后
- 每章完成后更新人物成长
- 根据反馈调整设定

### 文档 2：正文连载

**文件命名建议**：
```
[小说名]_正文_vX.md
```

**章节管理**：
- 使用明确的章节标题
- 每章开头标注完成时间
- 可在章节后添加作者备注

**更新策略**：
- 逐章追加
- 保留每章独立性
- 可标注关键剧情节点

## 与用户协作的建议

### 何时主动提问

1. 核心设定模糊不清
2. 存在明显逻辑矛盾
3. 风格方向需要确认
4. 剧情选择需要决策

### 何时只需推进

1. 信息基本完整
2. 用户对前期方案满意
3. 创作流程稳定
4. 反馈已明确

### 沟通方式

**推荐格式**：
- 明确分隔不同部分（使用---或##标题）
- 提供具体选项而非开放式问题
- 总结确认后再继续

**避免**：
- 一次性问太多问题（3-5 个核心问题为宜）
- 模糊不清的反馈
- 不解释修改原因

## 参考资源

### 结构参考
- 《救猫咪》三部曲 - 电影编剧结构，适用于小说
- 《故事》罗伯特·麦基 - 经典叙事理论
- 网络小说常见结构可参考起点中文网热门作品

### 人物塑造
- 角色动机要清晰
- 角色弱点比优点更重要
- 通过冲突展示性格

### 风格培养
- 阅读同类成功作品
- 分析其语言特点
- 模仿练习后形成自己的风格
